# AutoClip 自动切片核心工作流

## 概述

AutoClip 的自动切片工作流是一套完整的内容处理流水线，将长视频转换为适合短视频传播的精炼片段。整个流程分为**5个核心步骤**。

---

## 工作流总览

```
原始视频
   ↓
[Step 1] 文本预处理
   ↓
[Step 2] 大纲提取（主题识别 + 时长估算）
   ↓
[Step 3] 时间点定位（精确切分 + 时长验证）
   ↓
[Step 4] 综合评分（final_score + recommend_reason）
   ↓
[Step 5] 标题生成 + 主题聚类
   ↓
最终输出：可发布的短视频片段
```

---

## Step 1: 文本预处理

**输入**: 视频转写文本（已人工校对，保证基本语义连贯）

**输出**: 分块处理的文本段落（每段约30-60分钟）

**处理逻辑**:
- 按时间顺序切分文本
- 保持语义连贯性
- 为后续处理提供基础

---

## Step 2: 大纲提取（主题聚类.txt）

**输入**: 视频转写文本片段

**输出**: 结构化的话题大纲，包含每个话题的预计时长

**核心Prompt设计要点**:

| 要点 | 说明 |
|------|------|
| 完整覆盖优先 | 零遗漏要求：覆盖输入文本95%以上的有价值内容 |
| 数量控制 | 30分钟块提取2-5个核心话题；60分钟视频6-12个核心话题 |
| 时长估算 | 每个话题对应3-12分钟（目标6-8分钟） |
| 兜底机制 | 无法归类的重要内容创建"其他要点"收纳 |
| 开头结尾重视 | 视频前10-15分钟和后5-10分钟不可忽略 |

**话题提取原则**:
1. **重要性优先**: 核心话题优先，信息密度高的片段独立成话题
2. **时长平衡**: 避免过短（<2分钟）或过长（>15分钟）的话题
3. **结构完整**: 确保每个话题包含完整的讨论逻辑
4. **价值导向**: 跳过低价值内容（纯寒暄、口头禅）

**话题合并/拆分策略**:
- **合并**: 同主题不同角度、连续相关讨论、逻辑链条
- **拆分**: 话题转换明显、时间跨度大、受众价值不同

---

## Step 3: 时间点定位（时间点.txt）

**输入**: 大纲 + SRT字幕文本块

**输出**: 每个话题的精确 `start_time` 和 `end_time`

**核心Prompt设计要点**:

| 要点 | 说明 |
|------|------|
| start_time | 定位到话题"第一句核心发言"的开始，忽略寒暄/过渡 |
| end_time | 必须覆盖话题核心讨论的"最后一句话结束"，不可无脑设置为文本块结束 |
| 时长验证 | 每个片段必须 ≥90秒，<90秒需与相邻话题合并 |
| 目标时长 | 3-6分钟最佳，不超过8分钟 |

**语义边界识别**:
- **自然开始点**: 话题引入语句、观点转折、新案例引入
- **自然结束点**: 总结性语句、话题转换信号、观点收尾
- **避免切割点**: 句子中间、观点展开中、逻辑推理关键环节

**时长验证与合并流程**:
```
计算时长 → 检查<90秒 → 寻找可合并相邻话题 → 合并后验证≤8分钟
```

---

## Step 4: 综合评分（推荐理由.txt）

**输入**: 已定位时间的话题（包含outline、content、start_time、end_time）

**输出**: `final_score`（0.0-1.0）+ `recommend_reason`（15-30字）

**评分维度（四维评估）**:

| 维度 | 权重说明 |
|------|----------|
| 信息价值 | 独特见解/知识？信息密度高？ |
| 情感共鸣 | 引发强烈情感？观点鲜明？ |
| 传播潜力 | 有"金句"或"梗"？易引发讨论分享？ |
| 结构完整性 | 逻辑清晰、有始有终？ |

**推荐理由撰写要求**:
- 精准、诱人
- 体现话题最核心的亮点
- 15-30字

---

## Step 5: 标题生成（标题生成.txt）

**输入**: 高分片段列表（id、title、content、recommend_reason）

**输出**: 为每个片段生成一个最佳标题

**核心Prompt设计要点**:

| 要点 | 说明 |
|------|------|
| 忠于原文 | 立意必须直接源自片段内容，严禁无中生有 |
| 拒绝夸张 | 避免"震惊"、"惊呆了"等过度营销词汇 |
| 突出亮点 | 精准捕捉最核心观点、最激烈情绪、最有价值信息 |
| 精炼有力 | 简洁有冲击力，快速抓住用户眼球 |

**输出格式**: JSON对象，key为片段id，value为标题字符串

---

## Step 6: 主题聚类（主题聚类.txt）

**输入**: 所有视频切片（标题+摘要）

**输出**: 多个合集（collection_title、collection_summary、clips）

**聚类要求**:
- 每个合集至少2条切片，最多5条
- 考虑主题关键词（投资理财、职场成长、社会观察、文化差异等）
- 按主题相关性排序

---

## 分类型Prompt变体

针对不同内容类型，Prompt 有专门的变体设计：

| 类型 | 特殊考虑 |
|------|----------|
| **娱乐综艺** | 保持笑点完整、节奏自然、效果延续 |
| **商业财经** | 数据完整性、投资建议与风险提示配套 |
| **演讲脱口秀** | 表达完整性、情感连贯、观众反应保留 |

---

## 输出验证清单

每个步骤输出前必须验证：
- [ ] 时间点格式正确（HH:MM:SS,mmm）
- [ ] 时长符合要求（≥90秒，≤8分钟）
- [ ] JSON格式有效可解析
- [ ] 覆盖度检查（≥95%有价值内容）
- [ ] 语义边界自然完整

---

## 核心原则总结

> **完整覆盖优先，适度精炼为辅**
> 宁可多提取一个话题，也不要遗漏重要内容！
