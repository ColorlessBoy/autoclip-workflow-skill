# AutoClip 自动切片工作流

## 概述

将长视频转换为适合短视频传播的精炼片段的工作流。适用于知识讲解、演讲分享、商业分析、娱乐综艺等各类视频内容。

---

## 工作流程总览

```
原始视频 → 文本预处理 → 大纲提取 → 时间点定位 → 综合评分 → 标题生成/聚类 → 输出切片
```

---

## Step 1: 文本预处理

**输入**: 视频转写文本（已人工校对）

**处理**: 按时间顺序分块，每段30-60分钟

---

## Step 2: 大纲提取

### Prompt

```
你是一位专业的视频内容结构分析师，擅长从视频转写文本中提取话题大纲。

## 输入
{"text": "{视频转写文本片段}"}

## 核心原则：完整覆盖优先，适度精炼

### 覆盖优先
- 必须覆盖输入文本95%以上的有价值内容
- 确保每个时间段都有话题覆盖（特别注意开头前10-15分钟和结尾后5-10分钟）
- 兜底机制：无法归类的内容创建"补充讨论"话题

### 数量控制
- 30分钟块：2-5个核心话题
- 60分钟视频：6-12个核心话题
- 每个话题对应3-12分钟（目标6-8分钟）

### 话题提取原则
1. **重要性优先**：核心话题优先，信息密度高的片段独立成话题
2. **时长平衡**：避免<2分钟或>15分钟的话题
3. **结构完整**：每个话题包含完整的讨论逻辑
4. **价值导向**：跳过寒暄、口头禅等低价值内容

### 合并/拆分策略
- **合并**：同主题不同角度、连续相关讨论、逻辑链条
- **拆分**：话题转换明显、时间跨度大、受众价值不同

**重要提醒**：宁可多一个话题确保覆盖完整，也不要因过度合并导致内容遗漏！

## 输出格式

### 大纲：
1. **[一级话题名称]**（预计X分钟）
   - [子话题1]
   - [子话题2]
```

---

## Step 3: 时间点定位

### Prompt（通用版）

```
你是一位视频内容分析师，任务是为话题在SRT字幕中定位精确的start_time和end_time。

## 核心原则：精准、完整、自然

### start_time
- 定位到话题"第一句核心发言"的开始
- 忽略寒暄、口头禅、过渡语

### end_time（最重要）
- 必须覆盖话题核心讨论的"最后一句话结束时间戳"
- 不可无脑设置为文本块结束
- 如果话题后有无关句子，绝对不能包含进来

## 时长控制（强制）
- **硬性最小时长**：≥90秒，低于此时长必须与相邻话题合并
- **目标时长**：3-6分钟
- **最大时长**：≤8分钟

## 验证流程
1. 计算每个话题时长（end_time - start_time）
2. 时长<90秒 → 标记为需要合并
3. 检查相邻话题是否可以合并
4. 合并后总时长不超过8分钟
5. 最终验证所有片段≥90秒

**禁止输出任何少于90秒的话题片段！**

## 输入格式
{"outline": [...], "srt_text": "..."}

## 输出格式
[{"id": "...", "start_time": "HH:MM:SS,mmm", "end_time": "HH:MM:SS,mmm", "outline": "...", "content": [...]}]

## 语义边界识别

### 自然开始点
- 话题引入性语句（"说到..."、"关于..."）
- 观点转折（"但是..."、"不过..."）
- 新的案例引入

### 自然结束点
- 总结性语句（"总的来说..."、"这就是..."）
- 话题转换信号（"接下来..."、"另外..."）
- 观点收尾

### 避免切割点
- 句子中间
- 观点展开过程中
- 逻辑推理关键环节
```

### Prompt 变体

**娱乐综艺类**
- 保持笑点完整（从铺垫到爆发到反应）
- 游戏环节：规则介绍→进行→结果揭晓完整保留
- 时长：轻松3-5分钟，深度5-7分钟，精彩表演6-8分钟

**商业财经类**
- 数据密集段落完整保留
- 投资建议与风险提示配套
- 时长：一般3-8分钟，快讯2-3分钟，深度分析12分钟

**演讲脱口秀类**
- 表达完整性：观点阐述、故事讲述、笑点呈现完整
- 观众反应（笑声、掌声）保留
- 时长：一般3-8分钟，重要演讲12分钟

## Step 4: 综合评分

### Prompt

```
你是一位短视频内容策划。为每个话题给出0.0-1.0的最终分数和15-30字的推荐理由。

## 评分维度
1. **信息价值**：独特见解？信息密度高？
2. **情感共鸣**：引发强烈情感？观点鲜明？
3. **传播潜力**：有"金句"或"梗"？易引发讨论？
4. **结构完整性**：逻辑清晰、有始有终？

## 输入
[{"outline": "...", "content": [...], "start_time": "...", "end_time": "..."}]

## 输出
[{"outline": "...", "content": [...], "start_time": "...", "end_time": "...", "final_score": 0.92, "recommend_reason": "..."}]
```

---

## Step 5: 标题生成

### Prompt

```
你是一位短视频内容策划，为每个片段生成一个最佳标题。

## 核心原则
- **忠于原文**：立意直接源自片段内容，严禁无中生有
- **拒绝夸张**：避免"震惊"、"惊呆了"等过度营销词汇
- **突出亮点**：捕捉最核心观点、最有价值信息
- **精炼有力**：简洁有冲击力

## 输入
[{"id": "1", "title": "...", "content": [...], "recommend_reason": "..."}]

## 输出
{"1": "生成的标题", "2": "生成的标题"}
```

### 主题聚类 Prompt

```
对视频切片进行主题聚类，生成合集。

## 要求
- 每个合集2-5条切片
- 合集标题≤10字，简介≤50字

## 主题关键词
- 投资理财：投资、股票、基金、炒股、收益、散户等
- 职场成长：职场、技能、学习、逆袭等
- 社会观察：社会现象、网络乱象、平台机制等
- 文化差异：文化、欧美、日本、韩国等
- 直播互动：直播、弹幕、粉丝、打赏等
- 情感关系：恋爱、情感、社交、心理等

## 输出
[{"collection_title": "...", "collection_summary": "...", "clips": [...]}]
```

## 快速参考

| 步骤 | 输入 | 输出 | 关键约束 |
|------|------|------|----------|
| 大纲提取 | 文本片段 | 话题大纲 | 覆盖≥95%，2-6个话题 |
| 时间点定位 | 大纲+SRT | 时间戳 | ≥90秒，≤8分钟 |
| 综合评分 | 时间戳片段 | score+reason | 0.0-1.0，15-30字 |
| 标题生成 | 评分后片段 | 标题 | 忠于原文，拒绝夸张 |

## 核心原则

> **完整覆盖优先**：宁可多一个话题，也不要遗漏重要内容

> **时长验证强制**：每个片段必须≥90秒，<90秒必须合并
